---
description: "Draft a product spec / PRD with goals, user stories, acceptance criteria, and success metrics."
argument-hint: "[feature]"
---

Run the `product-spec` skill on `$ARGUMENTS`.

Turns an idea into a buildable, verifiable spec: problem statement, goals and non-goals, user stories, testable Given/When/Then acceptance criteria, success metrics with targets and windows, scope/phases, and open questions. Produces a document that feeds `verify-against-spec` and `complete-feature`.
