# Contributing to Apple Dev Skills

Thanks for your interest in contributing! This repo is a multi-platform skill set for Apple platform development. We welcome skills, bug fixes, platform support, and documentation improvements.

---

## Quick Start

1. Fork the repo
2. Clone your fork: `git clone https://github.com/YOURNAME/apple-dev-skills.git`
3. Make changes in `src/` only
4. Run validation: `node scripts/validate.js`
5. Run build: `node scripts/build.js`
6. Commit both `src/` and generated `platforms/` changes
7. Open a PR

---

## Golden Rules

1. **Edit only `src/`** — `platforms/`, `.claude-plugin/`, `.cursor-plugin/`, and `.kimi-plugin/` are generated by `scripts/build.js`. Never edit them directly.
2. **Validate before committing** — `node scripts/validate.js` must pass.
3. **Regenerate after edits** — After changing any `src/` file, run `node scripts/build.js` and commit both `src/` and `platforms/` changes.
4. **Keep skills self-contained** — Cross-reference other skills by name, not by file path.
5. **YAML frontmatter is required** — Every `SKILL.md`, agent, and command must have valid frontmatter.

---

## Adding a New Skill

1. Create `src/skills/<skill-name>/SKILL.md` with frontmatter:
   ```yaml
   ---
   name: skill-name
   description: Brief description for discovery
   ---
   ```
2. Add supporting files in subdirectories if needed.
3. Run `node scripts/validate.js`.
4. Run `node scripts/build.js`.
5. Commit `src/skills/<skill-name>/` and updated `platforms/`.

### Skill Content Guidelines

- **Self-contained** — A skill should make sense when loaded alone.
- **Severity framework** — Use P0 (Critical), P1 (High), P2 (Medium), P3 (Low) consistently.
- **Platform-neutral language** — Avoid "Claude Code will..." unless in agent definitions. Skills are consumed by multiple platforms.
- **Kimi adaptation** — If your skill dispatches subagents (e.g., `apple-cleanup`), add a "Kimi Adaptation" section with step-by-step manual instructions.

---

## Adding a New Platform

1. Add a builder function in `scripts/build.js`.
2. Add install path in `install.sh`.
3. Document in `PLATFORM_COMPATIBILITY.md` and `README.md`.
4. Add CI step in `.github/workflows/build.yml` if needed.

---

## Commit Message Style

```
feat(skill): add ios-accessibility audit patterns
fix(build): correct kimi script path in plugin.json
docs(readme): update install instructions for Cursor
ci(github): add release workflow
```

- `feat(skill)` — new skill or skill enhancement
- `fix(build)` — build script or platform output fix
- `docs` — documentation changes
- `ci` — GitHub Actions or automation

---

## Release Process

Maintainers only:

1. Ensure `main` is green (CI passes).
2. Bump version in `package.json`.
3. Run `node scripts/build.js` and commit.
4. Update `CHANGELOG.md`.
5. Tag: `git tag -a v0.1.0 -m "Release 0.1.0"`
6. Push tag: `git push origin v0.1.0`
7. GitHub Actions creates release with platform bundles.
