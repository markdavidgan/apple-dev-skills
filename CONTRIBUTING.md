# Feedback & Corrections

This repo is **maintainer-curated**. The skills contain Apple platform guidance that directly affects production code — API signatures, App Store compliance rules, Swift 6 concurrency patterns. The quality bar is intentionally high.

## What we welcome

- **Bug reports** — Outdated API signatures, broken install steps, typos
- **Corrections** — Wrong App Store guidance, unsafe concurrency advice, missing entitlements
- **Feature requests** — New platform support (Windsurf, Continue, etc.), new Apple frameworks
- **Forks** — Feel free to fork and adapt for your team or workflow

## What is maintainer-curated

- **New skills** — Open an issue to discuss before writing. New skills must pass the same crash-prevention and App Store compliance rigor as existing ones.
- **Core patterns** — Swift 6 concurrency rules, SwiftData safety patterns, and API signatures are authored or heavily reviewed by the maintainer.

## How to report

Open a [GitHub Issue](https://github.com/markdavidgan/apple-dev-skills/issues). Include:

- The specific skill or file
- What's wrong or missing
- A reference to official Apple documentation (if applicable)

## For maintainers

If you are Mark or a co-maintainer, see [docs/releasing.md](docs/releasing.md) for the release workflow.
